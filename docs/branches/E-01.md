## Исследовательский вопрос

- **VERIFIED FACT:** Канонический lineage: `S0 → A1 → M1 → A2 → M2 → A3 → M3 → A4 → M4 → FP8`.
- **ENGINEERING HYPOTHESIS:** Узко обученный E-01 с CodeWorld, policy-gated runtime и executable verification превысит general-purpose frontier APIs по Verified Enterprise Task Completion Rate на фиксированном наборе enterprise coding workflows при меньшей стоимости разрешённой задачи.
- **EXPERIMENT REQUIRED:** Сравнить frozen BF16 S0, M1, M2, M3 и полный M4 candidate на одном frozen private holdout; публиковать доверительные интервалы, latency и стоимость, а не один aggregate score.
- **RISK:** 480B/35B foundation может дать качество, но проиграть по cost per verified task и latency; без измеренного uplift и serving economics E-01 не проходит release gate.

## Work packages

1. **VERIFIED FACT:** E01-WP0 замораживает baseline, LÆTEX-Bench protocol и contamination registry до post-training.
2. **ENGINEERING HYPOTHESIS:** E01-WP1 фиксирует frozen BF16 S0 и обучает identity/tool LoRA; verified BF16 merge создаёт M1.
3. **ENGINEERING HYPOTHESIS:** E01-WP2 создаёт verified enterprise Action SFT trajectories и обучает второй LoRA; verified BF16 merge создаёт M2.
4. **ENGINEERING HYPOTHESIS:** E01-WP3 обучает retained preference adapter `A3`; отдельный verified BF16 merge создаёт immutable `M3=M2+A3`.
5. **ENGINEERING HYPOTHESIS:** E01-WP4 обучает retained executable GRPO adapter `A4`; отдельный verified BF16 merge создаёт master `M4=M3+A4`.
6. **ENGINEERING HYPOTHESIS:** E01-WP5 реализует CodeWorld V0, Organizational World Model V0 и policy-gated action runtime.
7. **EXPERIMENT REQUIRED:** E01-WP6 производит FP8 serving derivative только после parity с BF16 M4; internal router и experts остаются frozen до отдельного ablation gate.
8. **VERIFIED FACT:** E01-WP7 выпускает BF16 M4 master, parity-approved FP8 derivative, schemas, eval report, model card, SBOM, licenses и reproducibility manifest.

- **ENGINEERING HYPOTHESIS:** Broad CPT не входит в E-01 default work packages из-за alignment-loss risk.
- **EXPERIMENT REQUIRED:** Узкий domain CPT, router adaptation или expert adaptation допускаются только отдельной ветвью после доказанного adapter ceiling.

## Compute gate

- **ENGINEERING HYPOTHESIS:** Минимальная и рекомендуемая H200-конфигурации для 480B BF16 LoRA/SFT определяются только после exact shard inventory, ZeRO/FSDP/TP/EP memory model и representative profiling; прежняя оценка одного 8×H200 node для 80B foundation к E-01 неприменима.
- **EXPERIMENT REQUIRED:** До первого оплачиваемого run выполнить dry-run конфигурации без весов, проверить container digest, topology, NCCL/IB, encrypted object storage, checkpoint resume и telemetry.
- **RISK:** 480B BF16 weights занимают порядка 960 GB только для параметров без runtime/training overhead; фактический memory fit нельзя выводить из этой оценки без load test.
- **RISK:** Full-parameter training 480B запрещён до доказательства, что adapter ceiling блокирует release metric и что compute budget экономически оправдан.
- **VERIFIED FACT:** Checkpoints пишутся в tenant-isolated object storage, versioned registry и immutable lineage ledger; локальная выгрузка весов запрещена.

## Exit criteria

- **EXPERIMENT REQUIRED:** Candidate увеличивает VETCR относительно untouched foundation не менее чем на заранее зафиксированный minimum detectable effect без ухудшения security critical-failure rate.
- **EXPERIMENT REQUIRED:** Pass-to-pass regression на frozen coding suite не хуже frozen non-inferiority margin `<=2 pp`.
- **EXPERIMENT REQUIRED:** 100% write actions имеют policy decision, scoped credential, evidence bundle и replayable audit entry.
- **RISK:** При отсутствии statistically defensible uplift, устойчивого recovery после tool failures или приемлемой стоимости resolved task ветвь останавливается; identity-only rebrand не считается результатом.
