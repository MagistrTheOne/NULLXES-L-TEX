# E-01 — независимая Enterprise Action Model

## Status

- **VERIFIED FACT:** Branch decision: Accepted by ADR-0007.
- **VERIFIED FACT:** E-01 is independently initialized and pretrained from scratch; no external checkpoint or tokenizer is a parent.
- **VERIFIED FACT:** Qwen is benchmark and optional offline synthetic teacher only under ADR-0009.
- **VERIFIED FACT:** All model compute is NVIDIA H200-only; local environment is control/documentation plane without weights or training workloads.

## Target specification

| Field | Canonical status |
|---|---|
| Backbone | **ENGINEERING HYPOTHESIS:** decoder-only sparse MoE |
| Depth / width | **ENGINEERING HYPOTHESIS:** 64 layers / `d_model=8192` |
| Attention | **ENGINEERING HYPOTHESIS:** GQA `64Q/8KV`, head dimension 128 |
| Pattern | **ENGINEERING HYPOTHESIS:** 7 local + 1 global; local window 16 384 |
| Experts | **ENGINEERING HYPOTHESIS:** 144 routed + 1 shared; Top-6; expert `d_ff=2048` |

| Context | **ENGINEERING HYPOTHESIS:** 262 144 target, reached by curriculum |

| Scale | **EXPERIMENT REQUIRED:** `~478.9B total / ~34.4B active`, pending executable proxy validation |

## Research question

- **ENGINEERING HYPOTHESIS:** A scratch-trained, enterprise-specialized E-01 combined with CodeWorld, policy-gated runtime and executable verification can outperform general-purpose APIs on a frozen set of enterprise workflows by Verified Enterprise Task Completion Rate and auditability.
- **EXPERIMENT REQUIRED:** Compare E-01 checkpoints and external benchmarks on the same private holdout with confidence intervals, task-level evidence, latency and total H200 cost.
- **RISK:** Independent pretraining may fail to match coding quality, consume an uneconomic token/H200 budget or miss the context target; no superiority claim is currently verified.

## Work packages

1. **VERIFIED FACT:** E01-WP0 freezes evaluation, contamination registry and source/license policy before training data construction.
2. **ENGINEERING HYPOTHESIS:** E01-WP1 builds C0 corpus manifests and T0 custom tokenizer candidates.
3. **EXPERIMENT REQUIRED:** E01-WP2 runs parameter-count, dense/MoE and hybrid-attention proxies and produces signed architecture evidence.
4. **EXPERIMENT REQUIRED:** E01-WP3 validates H200 parallelism, checkpoint/recovery, storage and cost model.
5. **EXPERIMENT REQUIRED:** E01-WP4 issues G0 only when architecture, data, systems, safety and budget gates all pass.
6. **ENGINEERING HYPOTHESIS:** E01-WP5 executes full pretraining to immutable BF16 B0 with resumable checkpoints and frozen evals.
7. **ENGINEERING HYPOTHESIS:** E01-WP6 executes SFT, preference and executable-RL stages as separate lineage artifacts.
8. **EXPERIMENT REQUIRED:** E01-WP7 promotes BF16 M0 and serving derivative only after release/parity gates.

## Compute gate

- **EXPERIMENT REQUIRED:** Exact minimum/recommended H200 count and DP/TP/PP/EP/CP mapping remain unresolved until proxy memory/communication profiling.
- **VERIFIED FACT:** Eligible infrastructure is homogeneous production-grade H200 HGX with NVLink/NVSwitch and provider-attested InfiniBand.
- **EXPERIMENT REQUIRED:** Allocation must pass topology, NCCL, storage, failure recovery and checkpoint-resume acceptance before model run.
- **RISK:** `~478.9B` BF16 parameters imply roughly 958 GB parameter bytes before gradients, optimizer, activations, buffers and redundancy; this arithmetic is not a memory-fit proof.
- **RISK:** Full-scale launch without licensed data readiness and signed cost cap is prohibited.

## Exit criteria

- **EXPERIMENT REQUIRED:** Proxy parameter count matches the frozen config and active-path accounting within a predeclared tolerance.
- **EXPERIMENT REQUIRED:** Router stability, expert coverage, convergence and context curriculum pass frozen thresholds defined before each run.
- **EXPERIMENT REQUIRED:** M0 meets predeclared coding, tool, governance, safety and VETCR gates without critical regression.
- **EXPERIMENT REQUIRED:** 100% controlled writes carry policy decision, scoped credential, evidence bundle and replayable audit entry.
- **RISK:** Failure of scale transfer, data supply, H200 economics or recoverability returns ADR-0007 to review; external weight substitution is not an automatic fallback.
